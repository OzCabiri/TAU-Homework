When looking at the code we have a samll nop section before the 'printf' and a big nop section before the code that uses 'fgets'.

The small one will be 'patch1', and the big one will be 'patch2'.
Let's also denote the section that uses 'fgets' as 'get_line', and the section that uses 'printf' as 'print_line'.

The flow of the code means that we will encounter patch2 first, so the first line will be a jump to get_line.
The rest will check if the first 2 chars are "#!", if so it will push the command (starting address, just after '!') into the stack so the system's 'system' method can use it.
We then call system(), and jump to after printf.

patch1 will just contain a jump to the 2nd line of patch2 (we want to avoid the initial jump to get_line). Essentially a simple redirection.

Regarding the python code, we will just replace the bytes in the positions we talked about with patch1.asm and patch2.asm.
#===========================================#
Notes:
* I didn't want to use eax, because we might override it and it is still used right before printf.
* We'll use ecx for the line position in memory (closest thing to a counter right now).
* We'll use edx for storing and comparing to '#' and '!', becuase it's not supposed to be used.
* '#'=0x23, "!"=0x21.

Addresses arithmetic:

    patch2 1nd line address: 0x080485CD
    get_line address: 0x08048662
        patch2's 1st jump: 0x08048662 - 0x080485CD = 0x95
        
    patch2 starting address: 0x080485CD ***
    1st line after patch1 address: 0x08048638

    patch2 starting address: 0x080485CD ***
    line after printf in print_line address: 0x0804864E

    patch2 starting address: 0x080485CD ***
